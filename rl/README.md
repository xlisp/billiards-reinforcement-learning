# DQN 桌球 RL

让 DQN 自己学会打桌球——目标：**连续进球越快越多 reward 越高**。

环境基于 [`../docs/physics.md`](../docs/physics.md) 的物理（动量守恒 / 摩擦衰减 / 库边反射 / 进袋判定），奖励对应 [`../README.md`](../README.md) §4 §5 §9 的实战要义（效率、连击、容错）。

## 文件

| 文件 | 作用 |
|---|---|
| `billiards_env.py` | Gymnasium 环境：离散动作 `(角度, 力度)`，观测 = 球状态 |
| `dqn.py`           | Double-DQN：QNetwork + ReplayBuffer + DQNAgent |
| `train.py`         | 训练循环 + checkpoint |
| `play.py`          | 加载模型，pygame 可视化 |
| `brain_trainer.py` | **大脑模型自训练框架**：MoE（小专家） + 知识蒸馏 + 对抗者 + 课程学习——把"用小模型训练你大脑"做成可运行的 PyTorch 系统 |

## 安装

```bash
pip install -r requirements.txt
```

## 训练

```bash
cd rl
python train.py --episodes 5000 --balls 3
```

常用参数：

```
--balls 3              # 目标球数（cue ball 之外）
--episodes 5000
--device cpu           # 或 cuda / mps
--lr 3e-4
--gamma 0.97
--eps-decay 30000      # ε 线性从 1.0 衰减到 0.05 经历的环境步数
--save-every 500
```

训练日志示例：

```
[ep   200] R̄=+0.31  pocket̄=0.65/3  streak̄=0.40  shots̄=18.2  clear%= 4  ε=0.83
[ep  1000] R̄=+1.85  pocket̄=1.40/3  streak̄=1.10  shots̄=15.1  clear%=22  ε=0.51
[ep  3000] R̄=+4.10  pocket̄=2.40/3  streak̄=1.90  shots̄=11.5  clear%=58  ε=0.10
[ep  5000] R̄=+6.30  pocket̄=2.85/3  streak̄=2.55  shots̄= 8.7  clear%=85  ε=0.05
```

`R̄` 升高、`shots̄` 下降、`streak̄` 上升 = AI 越来越像高手。

## 设计要点

### 动作空间
- `Discrete(N_ANGLES * N_FORCES)` = 36 × 5 = 180
- 不建模旋转（高低杆 / 加塞）。spin 可以后续作为额外的离散维度加入。

### 观测空间
- `(n_balls, 3)` 拍平：每个球 `(x_norm, y_norm, alive)`
- 进袋的球位置归零 + alive=0

### 奖励（核心：实现"连续进球越快越多越好"）

| 事件 | reward |
|---|---|
| 进 1 颗目标球 | `+1.0 × (1 + 0.5×(streak-1))` |
| 一杆进 N 颗 | 上式 × N |
| 每出一杆 | `-0.1`（鼓励"快"）|
| 母球进袋 | `-3.0`（README §7 犯规惩罚）|
| 全清台 | `+5.0` |
| 失败一杆（没进球）| streak 清零 |

`streak` = 连续多少杆都进了球。从 README §9 来：高手单杆数高 = 连续不断杆。

### 物理简化
对应 `docs/physics.md` 的实现：
- §二 动量守恒 → 球-球弹性碰撞（沿法线方向交换动量）
- §三 摩擦 → 简化为线性减速 `μg`，**未建模旋转**（DQN 任务足够）
- §五 路径积分 → 欧拉积分到所有球停下
- §六 库边反射 → 法向反弹 + 切向能量损失

## 后续可改进

1. **加入旋转**：把 `hit_offset` 也作为离散动作维度，让 agent 学会用高低杆控制走位（README §1.2）。
2. **更稠密奖励**：靠近袋口时给 shaping reward，加速早期学习。
3. **Frame-stack**：拍多帧观测捕捉走位意图。
4. **PPO / SAC**：连续动作空间会更精细，但 DQN 已经足以让 agent 学到"瞄袋口"的几何直觉。
5. **课程学习**：先 1 球，逐步加到 5 球。

## brain_trainer.py：用小模型训练大脑大模型

把 README §9.3 / §17–§20 的"小模型→大模型"心法实现成神经网络架构。

```bash
python brain_trainer.py                    # 完整四阶段自训练
python brain_trainer.py --epochs 50        # 加大训练轮数
python brain_trainer.py --no-adversary     # 关闭对抗训练做对照
python brain_trainer.py --eval-only --ckpt brain.pt
```

**架构**：

| 组件 | 作用 | 神经网络 |
|---|---|---|
| `SkillExpert` | 小模型 = 单一技能（直线 / 薄球 / 反袋 / 解斯诺克 / 走位） | 32 维隐层小 MLP |
| `BrainModel`  | 大脑 = Mixture-of-Experts，用门控网络调用专家 + refine 头 | 64 维门控 + 64 维 refine |
| `Adversary`   | 自我设计的小生成网络，生成专攻大脑弱点的难题场景 | 32 维隐层 |
| `Curriculum`  | 阶段化组合：① expert 预训练 ② 蒸馏到大脑 ③ 弱点诊断与重训 ④ 对抗微调 | — |

**核心思想对应**：

- **"用小模型训练大脑大模型"** ↔ Stage 2 蒸馏：experts 已预训练好，把它们的知识蒸馏给大脑的门控 + refine 头。
- **"训练关键参数小模型去打败大模型"** ↔ Stage 4 对抗：Adversary 网络通过最大化大脑误差来生成难题，反过来训大脑。
- **"分解阶段化组合化训练"** ↔ Stage 1–4 的明确分离 + 各 expert 独立训练 + MoE 组合。
- **"自我设计小模型 引导训练大脑"** ↔ `self_design_reweight()`：评估每个场景类型的 MSE，找出最弱项，自动加倍训练那个 expert。
- **"现实迁移"** ↔ Gating 分布表（最后输出）说明大脑学会了"看局面调专家"——这就是 Be water (§18.2)。

## 用微积分思想看桌球：从离散一杆到连续梯度下降

桌球表面看是"一杆一杆"的离散事件，但**球的轨迹本身是连续的**，**连击的攒成也是一种累积**——这就给了我们用 dx/dt、∂/∂θ、∫ 来描述它的空间。再把这套连续描述塞进神经网络的损失函数，就回到了梯度下降。

### 1. 导数：速度、加速度、连续性

桌面上每一颗球的状态 `s(t) = (x, y, v_x, v_y)` 都是时间的函数：

```
位置一阶导:  dx/dt = v_x          ← 速度（球此刻往哪走）
速度一阶导:  dv/dt = -μg·v̂        ← 加速度（摩擦让它减速，docs/physics.md §三）
位置二阶导:  d²x/dt² = -μg·v̂_x    ← 等价的牛顿第二定律
```

**连续进攻的"导数"** 不是物理量，而是定义在"杆"这个离散索引 `n` 上的差分：

```
攻击势能:    ΔP/Δn  =  streak(n) - streak(n-1)        ← 这一杆相比上一杆"多续了几条命"
火力强度:    Δpocket/Δn                                ← 单位杆数进球的瞬时速率
熟练度爬坡:  ΔR̄/Δepoch                                 ← 学习曲线的斜率，即"AI 在变强的速度"
```

`R̄` 上升 ⇔ "训练这个函数"对 epoch 的导数为正；当导数 → 0，就是收敛。

### 2. 微分：一杆里"动一动"会怎样

一杆的结果是 `result = f(θ, F, hit_offset, layout)`：杆头方向 θ、力度 F、击球点 hit_offset，作用在当前局面 layout 上。**微分回答的是：抖一抖会怎样？**

```
∂result/∂θ           ← 角度抖 0.5° 进球还成立吗？这就是"容错率"（README §5）
∂(走位距离)/∂F       ← 力度多一格，母球多滚多远？这就是"走位灵敏度"（README §1.2）
∂result/∂hit_offset  ← 高低杆/加塞带来的方向修正
```

高手的本质：**寻找局部使得 |∂result/∂θ| 很小但 |∂(走位)/∂F| 又足够大的"鞍点式击球"**——容错高、控制强。蛮干派恰好相反：在 ∂result/∂θ 很大的奇点上击球，差 1° 就脱靶。

### 3. 积分：连击就是一种累积

把"一颗球"的奖励 `r(n)` 沿杆数 `n` 累积，就得到这一局的总价值。RL 里写作折扣回报：

```
G = Σ_{n=0}^{N} γ^n · r(n)        ← 离散积分（强化学习标准 return）
   ≈ ∫₀^T  e^{-λt} · r(t)  dt     ← 连续极限：一局是一条 r(t) 曲线，γ↔e^{-λΔt}
```

`shots̄` 下降 ⇔ T 变短；`streak̄` 上升 ⇔ 单位时间 r(t) 更密集；`R̄` 上升 ⇔ ∫r dt 变大。**README §9 的"连续不断杆"== 让 r(t) 曲线下面的面积最大化**——这就是变分问题。

球的轨迹本身也是积分：`x(T) = x(0) + ∫₀^T v(t) dt`，对应 `physics.md §五` 的欧拉积分（环境里就是 `dt = 1/60` 的累加）。

### 4. 离散桌球如何"平滑"，以接上梯度下降

DQN 看似离散——`Discrete(180)`、奖励是 step 函数、进球是 0/1 事件——但**梯度下降只能流过可微分的东西**。我们靠四种"平滑"把它接通：

**(a) 动作的平滑：argmax → softmax**
```
离散选择:    a* = argmax_a Q(s,a)               ← 不可导，∂a*/∂θ 处处为 0
温度软化:    π(a|s) = exp(Q(s,a)/τ) / Σ exp(·)  ← 处处可导，τ→0 还原 argmax
```
这就是 Boltzmann 探索 / SAC entropy 的几何意义：**给离散动作搓出一层连续概率皮**。

**(b) 奖励的平滑：阶跃函数 → shaping**
```
原始:    r = 1 if 进袋 else 0              ← 阶跃，∇r = δ 函数，几乎处处零梯度
shaping: r̃ = r + α·exp(-d²/2σ²)           ← d 是球到袋口距离，给"差一点"也留信号
```
这正是"后续可改进 §2"那条建议的微积分动机——**让 ∂r/∂a 在没进球时也非零**，否则 agent 在早期是在零梯度的沙漠里随机游走。

**(c) 状态的平滑：alive=0/1 → 软占用**
进袋的球可以保留为"概率 0.0、坐标轨迹延伸"的虚态，梯度仍能从布局结构流回去。

**(d) 策略的可导：值函数 → 策略梯度**
```
DQN 的 TD-loss:    L(θ) = E[(r + γ max_a' Q(s',a';θ⁻) - Q(s,a;θ))²]
                                            └── 这个 max 还是 argmax，θ 通过 Q 间接受梯度
策略梯度 (PPO/SAC):∇_θ J = E[∇_θ log π(a|s;θ) · Â(s,a)]
                                            └── 直接对 π 求导，连续动作 (θ_aim, F) 都能学
```

完整的接通链条：

```
真实球桌（连续物理）
      │  采样 dt
      ▼
环境（离散一杆= 一次 env.step）
      │  softmax + shaping
      ▼
可导损失 L(θ)         ← 把"打没打进"翻译成"误差有多大"
      │  反向传播 ∂L/∂θ
      ▼
∇_θ L
      │  优化器（Adam = 带动量的梯度下降）
      ▼
θ ← θ - η·∇_θ L      ← 这一步在做的事：让大脑下次打得更准
```

### 5. 在本仓库里的对应

| 微积分概念 | 代码位置 |
|---|---|
| `dx/dt`, `dv/dt`（连续轨迹） | `billiards_env.py` 的物理积分循环（欧拉法） |
| 折扣积分 G = Σγ^n r | `dqn.py` 的 TD target `r + γ·max Q(s')` |
| argmax → softmax | `DQNAgent.act` 的 ε-greedy（极端版的温度退火） |
| ∂L/∂θ → 梯度下降 | `optimizer.step()` 那一行 |
| 二阶差分 ΔR̄/Δepoch | 训练日志里 `R̄` 的爬坡 |
| 偏导 ∂result/∂θ（容错） | reward 设计里"差一点也算进步"——参见后续 shaping 改进 |

**一句话总结**：桌球是一系列"离散决策 → 连续运动 → 离散事件"的循环；强化学习把这循环包成 `(s,a,r,s')` 四元组，用 softmax 和 shaping 给它涂上一层可微分的皮，再让梯度下降在这层皮上滑下去——AI 学会打球，本质上就是 **∇_θ L 在解一个连续化之后的变分问题**。
